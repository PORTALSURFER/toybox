# Toybox + Patchbay-Gui Audit (2026-02-04)

## Executive Summary
This audit focused on correctness, realtime safety, and API clarity across `toybox` and `patchbay-gui`. The codebase is generally clean and well-documented, but a few issues affect correctness and robustness. The highest impact findings are a renderer upload padding bug in `patchbay-gui` and input-consumption behavior in dropdown widgets that can lead to multiple widgets handling the same click. Several additive API improvements will make downstream plugins less coupled to clack APIs while keeping compatibility.

## Findings (Ranked)
| Severity | Area | Finding | Impact | Location |
| --- | --- | --- | --- | --- |
| High | Renderer | Upload padding in `Renderer::upload` can reuse stale bytes when row alignment requires padding. | Visual corruption or nondeterministic artifacts, especially when sizes change or with partial updates. | `patchbay-gui/src/renderer.rs` |
| Medium | UI Input | `Ui::dropdown` reads `input.mouse_pressed` directly, bypassing input consumption; multiple widgets can react to one click. | Interaction glitches with multiple widgets in a frame. | `patchbay-gui/src/ui.rs` |
| Low | DSP Docs | `StereoComb` docs mention a “subtle offset,” but the code does not introduce one. | Documentation mismatch; potential user confusion. | `toybox/src/dsp/delay.rs` |
| Low | DSP API | `overlap_normalization` accepts arbitrary hop sizes without clarifying constraints. | Users may get unexpected normalization when hop size does not divide window size. | `toybox/src/dsp/window.rs` |

## RT Safety Notes
- `toybox` DSP helpers avoid allocations in the audio path.
- `AutomationQueue` uses a mutex; its usage remains on UI/control threads. No RT allocation was detected in audio thread helpers.
- `patchbay-gui` logging is file-based; it is used from GUI thread only, not audio.

## API Cleanup Opportunities
- Add `toybox::clap::prelude` for a stable import surface.
- Add checked variants for helpers that have implicit constraints (e.g., overlap normalization).
- Re-export `hz_to_samples` alongside `ms_to_samples` to keep the DSP prelude consistent.

## Test Gaps / Suggestions
- Add a renderer padding test to ensure padded rows are zeroed.
- Add a dropdown input-consumption test to prevent multi-widget click conflicts.
- Add tests for new checked overlap normalization behavior.

## Proposed Fixes (Implemented)
- Zero-out upload padding in `Renderer::upload`.
- Use input-consumption aware `mouse_pressed()` in dropdown handling.
- Correct `StereoComb` documentation.
- Add `overlap_normalization_checked` for explicit constraints.
- Add `toybox::clap::prelude` to reduce direct clack dependency in plugins.

## Migration Notes (Additive)
- Existing APIs remain; new helpers are additive.
- Example plugin updated to use `toybox::clap::prelude`.
